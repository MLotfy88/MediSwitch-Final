## 🚀 نقاط التركيز الحالية (Active Focus)

تم بنجاح الانتهاء من حزمة الإصلاحات الشاملة للوحة التحكم (Admin Dashboard) والـ Cloudflare Worker:

### 1. تكامل لوحة التحكم وتصحيح الكود (Admin Dashboard Integration & Fixes)
*   **تم حل خطأ الـ Feedback Hub (500)**: بإضافة دوال `handleGetFeedback` و `handleUpdateFeedback` في `worker.js`.
*   **قياس أداء الـ API الفعلي**: استبدال قيمة API Latency الوهمية (42ms) بحساب زمن الاستجابة الفعلي لطلبات لوحة التحكم.
*   **إزالة البيانات الوهمية (User Intelligence)**: ربط الشاشة بقاعدة البيانات الحقيقية واستخدام هيكلية الحقول الفعلية مع تمكين أدوات Access Control لتجميد/تفعيل الحسابات حياً.
*   **بوابات الميزات وباقات الاشتراك**: إنشاء صفحة مخصصة مستقلة `Subscription Plans` لإدارة باقات الاشتراك (IAP Products) وتحديد صلاحيات كل باقة (Features Gating) وتفعيلها/تعطيلها.
*   **تفعيل Campaign Commander**: ربط أزرار الإجراءات (حذف الحملة ومعاينتها) مع بناء محاكي الإشعارات (Simulated Phone Notification) للمعاينة البصرية.

### 2. الربط الكامل للتهيئة والإعلانات والاشتراكات (Ads & Subscriptions Wiring)
*   **بوابات الميزات (Feature Gating)**: الربط البرمجي لملفات التطبيق مع صلاحيات الاشتراك.
    *   تطبيق فحص `SubscriptionProvider.isPremiumUser` لتعطيل الإعلانات تماماً (Banner و Interstitial) في كافة الشاشات تلقائياً عند تفعيل الباقة المميزة.
    *   التحكم في الميزات الأخرى (مثل: عدد الأدوية المتاح فحصها في فاحص التفاعلات، تفعيل المفضلة غير المحدودة، وتصدير التقارير لـ PDF).
*   **التحقق السحابي للاشتراكات (Subscription Validation)**:
    *   التطبيق يقوم بالاتصال بـ `$_baseUrl/api/v1/subscriptions/validate/` للتحقق من التوكنات، ولكن الـ Worker السحابي لا يحتوي حالياً على هذا المسار.
    *   *المطلوب*: تعديل وإضافة مسار Verification عام ومحمي برمز بسيط في `worker.js` لتوصيل عملية الشراء بقاعدة البيانات السحابية `subscriptions` وتغيير حالة المستخدم برمجياً.

### 3. اختبار المزامنة اليدوية والتلقائية (Sync Testing)
*   التحقق من عمل زر "مزامنة البيانات" محلياً في الإعدادات واكتشاف التعديلات الفروقية الجديدة للأدوية والأسعار بشكل آمن يدوياً وأوتوماتيكياً.

## 💡 القرارات النشطة (Active Decisions)

*   **ربط الإعلانات بالـ Provider**: تعطيل عرض الإعلانات برمجياً في `BannerAdWidget` و `AdService` عند رصد حساب مميز.
*   **استخدام الـ SQLite المحتفظ به للتفاعلات المجلوبة**: إبقاء التفاعلات التي تم استرجاعها من الإنترنت سابقاً مخزنة محلياً بالكامل لتعزيز الأداء والعمل أوفلاين.

## 🚧 التحديات والمخاطر المتبقية (Remaining Challenges)

*   **اختبار الشراء دون متجر حقيقي**: اختبار ميزة `in-app-purchase` يتطلب تهيئة أدوات الفحص المحلي في محاكي الأندرويد وجوجل بلاي (Sandbox).
*   **أمن الـ API**: خلو الـ API الخلفي للـ Worker من فحص التوكنات الحقيقي لعمليات المزامنة والتعديل (باستثناء تحديثات GitHub) يمثل ثغرة يجب تأمينها برمز وصول (Bearer Token) موحد ومحمي.
